# tabletop_access_ycb — design notes

A clean fork of `tabletop_access:access` (Ait Bouhsain et al., HAL 2025 —
the free-standing 3-tier shelf): retrieve an object-of-interest (OoI)
from a cluttered middle shelf and deliver it to the top deck.  **The
simulation environment is the same shelf**; what changes is the object
set and the grid:

* **Real GSO/YCB meshes** replace the uniform placeholder boxes.
* **Finer 3 cm grid** with **multi-cell rectangular footprints** — bigger
  objects occupy more cells, enabling tighter arrangements than the 6 cm
  single-cell parent.
* **Per-footprint-size `pick`/`put` actions** so multi-cell occupancy
  updates correctly.
* **Two placement regions** — `middle_deck` (pick/relocate) and
  `top_deck` (OoI goal).  The bottom floor compartments are dropped.

No existing `tabletop_access` code is modified; this is a sibling package.

## Why two regions

The floor compartments are cramped under-deck (~5 cm z-window) and carry
the parent's documented *cross-region put basin coupling* failures
(`REACHABILITY_METHODOLOGY.md`: `middle → floor_left ❌`).  Dropping them
removes that fragility; middle→top is the paper's core retrieval task and
240 cells (120/region) give ample relocation scratch.  It also trims ~20%
of the grid (a secondary benefit — the dynamic per-problem cell roster is
the real PDDL-size lever).

| `e_lego_duplo` | (same) | 2×4 |

| `orange` | (same) | 3×3 |
| `potted_meat_can` | (same) | 3×4 |
| `master_chef_can` | (same) | 4×4 |

**Excluded** (with reason): `mustard_bottle` — FAST-only (tall tapered;
FULL put-to-top fails); `mug`/`pitcher_base` — protruding handles;

### Mass / inertia

Cached YCB MJCFs derive mass from mesh volume × default density →
0.09–5.7 kg, which makes contacts violent.  `apply_runtime_tweaks`
overrides every movable to **mass 0.05 kg + diaginertia
(0.025, 0.025, 0.0125) + identity iquat** — matching the old placeholder
boxes (light, contact-gentle, tip-resistant).  Set at runtime on
`model.body_mass`/`body_inertia` (read by `mj_crb` each step); no
shared-infra edits, no MJCF mangling.

## Footprints (`footprint.py`)

An object's footprint is the set of grid cells its **collision
cross-section actually overlaps** (projected at the canonical grasp yaw,
inflated by a 6 mm place-clearance).  Invariant: a cell left `empty` is
genuinely clear of the object.

* **Round vs rectangle.** The overlap computation frees bounding-box
  corner cells for round objects — but *only* once an object spans ≳5–7
  cells across (a disc covers its corner cells otherwise).  At the chosen
  3 cm grid every roster object is 2–4 cells wide, so **all footprints are
  full rectangles**.  The round code stays; it auto-engages if the grid is
  ever refined to ~2 cm.  (Decision recorded after an empirical sweep
  across 2/2.5/3 cm.)
* **Canonical yaw** aligns the object's narrower horizontal extent with
  world-x (the FRONT-grasp finger axis) so the gripper fits.
* **Canonical pose** places the cross-section centre at the bounding
  block's centroid, resting on the deck.  `place_pose` derives the body
  origin from the measured cross-section centre + bottom-z because mesh
  body-origins are NOT AABB-centred (a ±20 mm error otherwise).

## Workspace (`env_builder.py`)

Identical `MultiTierShelf` to the parent `access` (deck_size (0.60, 0.45),
deck_levels (0.30, 0.55), legs, floor separator), at `shelf_pos
(0.25, 0.70, 0)`.  Two `GridRegion`s (`middle_deck`, `top_deck`),
`cell_size = 0.03`, **`level_z = deck surface`** (the parent baked a
0.05 m item-half reference into `level_z`; here placement adds each
object's own half-height).  12×10 cells each; `excluded_cells` empty.

## PDDL (`pddl/domain.pddl`, generated by `pddl_gen.py`)

Domain `tabletop-access-ycb`.  Pure STRIPS + typing (no
negative/derived/existential → pyperplan- and pymimir-friendly; single
file; concrete `(occupied …)` goals).

* Types `cell`, `movable`, `direction`; constants `north east`.
* Fluents `(occupied ?c ?o)`, `(empty ?c)`, `(holding ?o)`,
  `(gripper-empty)`; static `(adjacent ?dir ?c1 ?c2)` (the GNN spatial
  signal — pick/put use it only as rigid-block glue) and one
  `(fp_<W>x<H> ?o)` marker per footprint size.
* **Per-size schemas** `pick_<W>x<H>` / `put_<W>x<H>`: the W·H cell
  parameters are pinned into a rigid contiguous block by
  `(adjacent east/north …)` static preconditions, so a put/pick flips
  occupancy for *every* covered cell.  Default roster → sizes
  {2×2, 2×3, 2×4, 3×3, 3×4, 4×4} → 12 schemas.  Fast Downward grounds and
  solves these; the static rigid-shape preconditions keep grounding
  bounded (≈ #anchors per size, not W·H-combinatorial).

The canonical geometric pose for an action is the centroid of its cell
parameters; the bridge/state layer owns that mapping, so the chains stay
footprint-agnostic.

## Bridge (`bridge.py`)

`make_tabletop_access_ycb_bridge` — code predicates `occupied`/`empty`
evaluated from each object's footprint at its current pose (multi-cell;
position-memoised), `holding`/`gripper-empty` fluents, and per-size
`pick_<W>x<H>`/`put_<W>x<H>` executors that map the cell params → SW
anchor + centroid → chain `pick_fn`/`put_fn` and flip occupancy for all
covered cells.  The static `adjacent`/`fp_*` predicates are NOT bridge
predicates (enumerating `adjacent` over all cell pairs would blow up
`ground_state`) — they're emitted in the problem `:init` by `pddl_gen`.
The fork plans with the custom planner, not bridge/UP planning.

## State restore (`state.py`)

`restore_state` — canonical, history-independent: each object's anchor is
recovered from its `(occupied …)` facts (SW corner of the occupied
cells), placed at the footprint centroid via `place_pose`; velocities
zeroed, arm at staging-home; a `holding` object attached at the canonical
grasp offset.  FAST==FULL only holds under this restore.

## Chains (`chains.py`)

`make_ycb_access_chains` — forked from the parent `make_access_chains`,
adapted to 2 regions + the `level_z = surface` convention.  FRONT palm-+y
grasp; per-level hand-off teleport; **centre-grasp** at the object's COM
(tall items don't tilt on lift); variable grasp-z read live from each
object's half-height, clamped to the region's z-window; captured held-z
offset so cross-region puts land on the surface; FAST mode collapses
row-step lerps to 1 substep.  Footprint-agnostic (grasps/places at the
centroid the caller passes).

## Feasibility (`feasibility.py`)

`check_action` / `check_action_sequence` derived from the chain (feasible
⇔ chain succeeds), footprints threaded, FAST teleport + IK iter cap (80).
Footprint-aware structural prefilters `footprint_blocks_pick` /
`footprint_blocks_put` (corridor occlusion over cell-sets) short-circuit
the planner's candidate ordering without crashing.

**FAST==FULL: 11/11** roster objects (pick-middle + put-top), validated by
`examples/ta_ycb_validate.py --feasibility`.

**Dense-scene stress** (`examples/ta_ycb_stress.py`, 30 dense layouts /
120 per-action probes): **97.5% FAST==FULL agreement**, with ~1.7%
**FAST-accept / FULL-reject** on picks (narrow but real — dense occlusion
fools the optimistic teleport).  Timing (median): FAST 70–165 ms vs FULL
~3.4 s feasible / ~1.5 s infeasible → **20–30× speedup**; FAST per-check
(~100 ms) is the rgnet-relevant number.  Consequence: a short plan has a
~7–13% chance of containing a FAST-only action, so data-gen FULL-validates
**every** plan (see below) rather than spot-checking.

## Reachability

`examples/ta_ycb_validate.py --reachmap` (FAST): `middle_deck` pick 80/80
anchors, `top_deck` put 79/80 (only the extreme back-right anchor fails).
The per-action feasibility check gates that corner, so `excluded_cells`
stays empty — matching the sister domains' feasibility-gated convention.

## Planner (`planner.py`)

`solve` — feasibility-guided greedy clear-and-deliver (footprint-aware,
2-region), identical in spirit to the parent: relocate pickable blockers
that occlude the OoI pick or sit on/block the goal to feasibility-verified
scratch anchors, then pick the OoI and put it at the goal.  Every action
is FAST-feasible by construction.  `make_fast_oracle(setup)` wraps
restore + FAST `check_action`.

## Data generation (`generate_data.py`)

Samples cluttered middle-deck layouts (OoI + occluders in its front
corridor + clutter, packed non-overlapping), solves each, writes a PDDL
problem + a domain-matching **grounded** plan
(`grounded_plan_action`).  Soundness: per-action FAST checks (canonical
restore) build the plan, then **every plan is re-validated under FULL**
(`--full-check-frac` default 1.0, reject+resample on failure) so every
written plan is FULL-executable — the dense-scene ~1.7% FAST-optimism
makes a 10% spot-check insufficient, and full validation is cheap because
plans are short.  Problems emit a **dynamic per-problem cell roster**
(plan-touched cells + 1-ring margin, ≈100 cells) — not the full 240 — plus
within-region adjacency and per-object `fp_*` markers.  Difficulty axis:
packing density (occluders + clutter counts; `eval_ood` denser).

Cost with the full gate ≈ 60 s/instance single-process (FAST gen + one
FULL plan validation + resamples); the real run belongs on a
workstation/SLURM.  A parallel feasibility pool (cf. the parent
`access19/parallel.py`) is the obvious speedup and is not yet ported.

Generate:
```
python -m tampanda.symbolic.domains.tabletop_access_ycb.generate_data \
    --output-dir data/access_ycb --train 300 --val 50 --eval 50
# fast local iteration (no FULL gate -> plans only FAST-verified):
#   ... --train 20 --val 0 --eval 0 --full-check-frac 0
```
